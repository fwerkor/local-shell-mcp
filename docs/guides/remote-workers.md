# Remote workers

Remote workers let `local-shell-mcp` control machines that can make outbound HTTP(S) requests but cannot accept inbound SSH connections.

Typical use cases:

- HPC clusters.
- NPU/GPU servers behind NAT.
- Firewalled lab machines.
- Temporary build hosts.
- Machines where SSH setup is inconvenient but outbound HTTPS is allowed.

## Concept

```text
ChatGPT -> local-shell-mcp control server -> outbound polling worker -> remote machine
```

The worker joins with a one-time invite and polls the control server for jobs. The control server exposes `remote_*` tools to the MCP client.

## Basic workflow

1. Create an invite with `remote_invite`.
2. Copy the generated command.
3. Run it on the remote machine.
4. Confirm the machine appears in `remote_list_machines`.
5. Use remote tools such as `remote_run_shell_tool`, `remote_read_file`, and `remote_push_file`.
6. Revoke the worker when done.

## Capabilities

Remote tools cover near-parity operations:

- Shell and persistent shell.
- Filesystem read/write/search/patch.
- File transfer between local and remote workspaces.
- Git operations.
- Playwright/browser utilities where dependencies exist on the remote machine.

## Security model

A joined remote worker gives the MCP client control over the worker's configured environment. Use disposable accounts or dedicated directories when possible.

Recommended practices:

- Use short invite TTLs.
- Revoke workers after the task.
- Do not attach machines with unrelated production secrets.
- Avoid running as root unless the whole machine is disposable.
- Keep remote logs and control-server audit logs for review.

## Version matching

Remote worker code should match the control server version. Prefer the command generated by `remote_invite`, because the control server can provide the correct worker bootstrap.

## Troubleshooting

If a worker does not appear:

- Check outbound HTTPS access from the remote machine.
- Confirm the public base URL is reachable from the worker.
- Confirm the invite is not expired.
- Check system time on the remote machine.
- Inspect control-server logs.
