# Code of Conduct

This project follows a practical contributor standard: be respectful, stay technical, and keep discussions focused on improving the project.

## Expected behavior

- Be direct but professional.
- Discuss ideas, code, and evidence rather than personal traits.
- Keep issue and pull request threads on topic.
- Assume good faith while still requiring tests and clear reasoning.
- Respect security-sensitive disclosure boundaries.

## Unacceptable behavior

- Harassment, threats, or targeted personal attacks.
- Publishing private credentials, exploit details, or personal information.
- Repeatedly derailing technical threads.
- Abusive language toward maintainers, contributors, or users.

## Enforcement

Maintainers may edit, hide, lock, or delete disruptive comments; close issues or pull requests; restrict participation; or escalate to GitHub moderation when necessary.

Report conduct issues through the repository owner's public GitHub contact path or by opening a private security advisory when sensitive information is involved.
